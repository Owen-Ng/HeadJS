Head.js

The main purpose of Head is to provide a more unique and fun way of displaying informations and navigate between different screens.

Getting started!!!

Step 1.

You will need to click on "Download"(Top right).

Step2.

Then place it somewhere where it will be easily accessible for your html files. You will need 3 files, the downloaded files, a js file to manipulate the jslibrary and your html file.

Step3.

The following will be enough to get you started


main.js (manipulate the library):

				
	"use strict"; 

	function examples() {
		const head =new Head([150,900], 200, "/image/jslogo.JPG",202, true, "duplicating ");
		 //where you instantiate or start the library
			
		 head.addanimationbyid("anim", "coloring #0F83A9 red",500); //additional method that you can use
		 head.addanimationbyid("anim1", "coloring #0F83A9 grey",500);//additional method that you can use
		 head.addanimationbyid("anim2", "coloring #0F83A9 red",500);//additional method that you can use
	}

	examples();
	//The path "/image/jslogo.JPG" may not be exactly the same as yours. The path depends on where you placed the file so 
	//treat it as a regular file path
	//This example is from the landing page

html file (Make sure that the jsfile that will manipulate the head.js run after the library. Like the following):

				
		<head>
		<script defer type="text/javascript" src='/js/head.js'></script>
		<script defer type="text/javascript" src='/js/api.js'></script>
		</head>
		//The path "/js/head.js"or "/js/api.js"  may not be exactly the same as yours. The path depends on 
		//where you placed the file so treat it as a regular file path
		//head.js - is the library
		//api.js - is the one that manipulate the library.

Head(startcoord, diameter, backimg, backimgsize, draggable, animation)

Initialize the js object library. Creating a widget with functionalities that you intend to make.


Options

Param Default Type Decription
startcoord / required Array of number (size 2) initialize where you want the widget to start
diameter / Number The size of the widget
backimg / String It is the file path of the background image.
backimgsize / Number The size of the backimg
draggable false Boolean Do you want the widget to be draggable?
animation / String Animation:
"duplicating {Number}" When moved, there will be the backimg that will duplicate itself. {Number} (optional default at 70) the higher the number the longer will be the animation.

"iconimg {String} {Number}s {Number}t" when moved, the {String} image path will be used as animation when dragged. {Number}s optional the size of the icon image. Note: {Number}s and {Number}t are both numbers do not include 's' and 't' after a number. They are there to differentitate between the 2. {Number}t (optional default at 70) the higher the number the longer will be the animation.

"shadowing {String}" When moved and clicked, there will a shadow depending on how you implement it. For more reference on how to use shadowing "https://www.w3schools.com/cssref/css3_pr_box-shadow.asp"
If you want more freedom on the appearance of the widget, the head and the animation widget have their own ids namely 'head' and 'headclone' respectively
			
		#head:{ 
		
			
			e.g, border-radius: 0px !important; //This will change the widget to a square
		}
		#headclone:{
			e.g, z-index: 10 !important; // This will change the depth of the animation
			
		}
		//Since you will be overwritting, the initial design you will have to add 
				//!important after assigning a change in the appearance

Method:

addoubleclick(link)

Method to implement double click navigation between different screens set by the user.Double click will navigate to the next page set by the user.
Param Default Type Decription
link / required Array of String The string are the html name. Make sure all html pages are on the same folder

addclicklist(listarray, W, L, backgroundcolor, onmouseanimation, animationtime)

Method to implement a click event that will display a list when clicked. The list will adjust itself depending on the length and width provided. If an Shadowing was added as animation in the initialize object of Head(), The shadow will show up around the widget when clicked.
Param Default Type Decription
listarray / required Objects of Array Provide which html pages must have a specific list.
If you just want to display a message, you can have a simple string as the element but should not start with 'link'. e.g 'anything you want'.
If you want to navigate between other screens, it should first consist of 'link', the html file and then the name of the element('link {html file} {name}').
e.g 'link Screen.html page 1'
e.g {'Screen.html':['hello', 'link Screen1.html page1'], 'Screen1.html': ['Yes','link Screen page' ]}
W / Number width of the elements of the list
L / Number length of the elements of the list
backgroundcolor / String The color of the list. You can use HEX color
onmouseanimation / String Choose between 'highlighting' and 'coloring'. For coloring, 'coloring {String}' The {String} is the color you choose can either be a color or an hex code.
animationtime 1000 Number The time it takes in millisecond for the animation transition.
If you want more freedom on the appearance of the list, each list has been assigned an id respectively. Starting from l0 to ln, wherebye n is a natural number.
			
	head.addclicklist({'Screen1.html':['Screen1', 'how are you', 'OKKKKKK','link Screen3.html Screen3',
	 'link Screen2.html Screen2'],'Screen2.html':['Hello']},100,20, 'white', 'coloring orange');
For example, the above code point at 2 screen, Screen1 and Screen 2. Each of them has a list of array of either a link or a message. If you want to modify individually the appearance of each specific element of the list. You will have to create a css file like the following:
			
		#l0:{ 
		//This one will make changes to the first element of the list.
		//For example, 'Screen1.html' will be for 'Screen1' and 'Screen2.html' will be for 'Hello'
			//e.g, z-index: 10 !important;
		}
		#l1:{
		//This one will make changes to the Second element of the list.
		//For example, 'Screen1.html' will be for 'how are you' and 'Screen2.html' will be nothing 
		//because it contains only one element.
		}
		//Since you will be overwritting, the initial design you will have to add 
				//!important after assigning a change in the appearance

addanimationbyid(id, animation)

A seperate function method to animate text outside of the widget given an id and animation type.
Param Default Type Decription
id / required String The id of the element you want to animate.
animation /required String Type of animation 'coloring', 'shadowing' and 'highlighting'.
For coloring string format, 'coloring {String}' the {String} is the color that can be an HEX code or a color name.
For the shadowing string format, 'shadowing {String}' the {String} is a shadow box property
transitiontime 1000 Number The transition time for the animation.

addtouchhold(listarray, backcolor,width, height, onmouseanimation, mouseleavetime, transtime)

Method to display buttons that will navigate between screens. Mininmum of 1 and a maximum of 4 buttons can be created. When your mouse is on hold on the widget, The buttons that is been listed will show up. This method is used to navigate within the page.
Param Default Type Decription
listarray / required Object of array The objects are the html pages and the list are the link to those buttons. e.g {'Screen1': ['top', 'bottom'], 'Screen4.html': ['top', 'bottom','left', 'right']}
backcolor / String the background color.
width / Number width of the button.
height / Number height of the button.
onmouseanimation / String Type of animation 'coloring' and 'highlighting'. For coloring, 'coloring {String}' the {String} is the color can be an HEX code or a color name
mouseleavetime 1000 Number The time in millisecond for the button to disappear. Default 1000.
transtime 500 Number The time it took for the button animation.
If you want more freedom on the appearance of the button, each button has been assigned an id respectively. Starting from b0 to b3
			
	 head.addtouchhold({'Screen3.html':['top', 'down', 'rightttt', 'lefttttttttttttt']},'','','white', 'coloring orange')
For example, the above code point at Screen3. It has a list of array of ids that is within the page. If you want to modify individually the appearance of each of the button. You will have to create a css file like the following:
			
		#b0:{ 
		//This one will make changes to the first element of the list.
		//For example, 'Screen3.html' will be for 'top' 
			e.g, z-index: 10 !important;
		}
		#b1:{
		//This one will make changes to the Second element of the list.
		//For example, 'Screen1.html' will be for down'
		}
		//Since you will be overwritting, the initial design you will have to add 
				//!important after assigning a change in the appearance
